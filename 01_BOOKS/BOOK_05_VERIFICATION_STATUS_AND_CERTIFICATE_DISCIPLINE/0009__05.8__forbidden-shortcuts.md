The purpose of Book 05 is to make these shortcuts impossible to miss. Each entry above is one *local* shortcut — a single object promoted past its license. The table is necessary but not sufficient: a reviewer can clear every row and still smuggle the deepest shortcut of all, which is to pick a structure because it *fits* rather than because it is *forced*. The verification principle below closes that gap.

### 05.8a Classification principle (derived-not-fitted)

This is the methodological law that stands behind the whole table above. State it as a verification rule, not as a slogan.

> **Verification principle (FORCED).** Every fundamental object in D0 is admitted only as a *classification* or *existence/uniqueness* theorem. Optimisation, curve-fitting, best-of-N selection, and post-hoc choice are categorically disallowed as closure mechanisms for a CORE object. A number that *fits* is not a number that is *forced*.

**Why this is forced, not stylistic [^b05-23].** The family of M1-equivalent extensions of any construction is infinite (GOLDEN EX 0.4.3), so an argument of the form "variant 1 fails, variant 2 fails, ... therefore mine is right" never terminates and never closes a theorem. The only finite route is to forbid every add-on that needs an external specification and **take the only admissible object, not the best-scoring one** [^b05-24]. Minimality and uniqueness in this corpus are therefore instances of one reusable template [^b05-25], never standalone assertions. This is the field form of the hostile-reading question: a "yes, a different admissible expression would also pass" means selection survived and the object is *not* closed; a "no" means it was forced. The §05.6 hostile-uniqueness controls (alternate projector, alternate irrational, zero-padded/random carrier, label permutation) are exactly the executable test of this principle — they are how a reviewer checks that the "no" is real rather than asserted.

**The canonical objects, and where each is forced (cite owners; do not re-derive here).**

The forbidden reflex — "a selected number written in a table ⇒ finite selector theorem" — is row 18 of the forbidden-shortcuts table above; this principle is the positive statement of the same firewall and the two must be read together.

**Verification obligation.** The general principle is closed (it reduces to GOLDEN LEM 0.4.4 / THE 0.4.5). The *per-object* executable witness — a certificate that mechanically rules out the nearby fitted alternatives for `K(9,11,13)` and for the memoryless-detector branching law — is **[^b05-22]**: a "minimal readout carrier" classification certificate and a "memoryless detector automaton" certificate are named as obligations, but no such certificate yet exists, so none is cited here. A shortcut that survives every row of the table but fails this principle — an object that merely *fits* — is still a forbidden shortcut. It is the one the table cannot catch, and the one this section exists to name.

### 05.8.R The audit rule: absence of alternatives is not an argument

This firewall has a mirror image that governs **reviews**, not just claims, and it is binding on every internal critique of D0. A critique of the form *"this number is suspect because one can enumerate other small expressions that would also fit"* is **not a valid argument** — it is the look-elsewhere reflex, and M1 forbids it in exactly the way it forbids the fitted claim: enumeration of what *could* have been does not bear on what the structure *forces*. A review may lower a claim's status on only two honest grounds:

1. **forcing-uniqueness fails** — the structure is shown to admit more than one object, named explicitly (then, and only then, the claim drops to a named-gap with the missing forcing stated); or
2. a **named gap** is already open — a specific, unfilled forcing obligation.

The valid question is never *"how many other numbers would fit?"* but *"does the structure admit exactly one?"*. The first is enumeration (forbidden); the second is forcing-uniqueness (required). A near-return inside a **structural grammar of invariants** verified to be minimal (e.g. the first continued-fraction convergent expressible in the named invariants) is forcing, not look-elsewhere — the distinction is the grammar: minimality among *named structural expressions* is a theorem, minimality among *arbitrary integers* is a fit. When a critique cannot exhibit a second forced object and cannot name a gap, it has produced no finding. This rule keeps the review layer under the same discipline as the theory.

The mirror obligation is symmetric: the rule licenses *refusals* exactly as strictly as it licenses *claims*. The worked refusals of the external-theorem audit — Leech `Λ₂₄ ↔ K=30`, `Spin(8)` triality ⇒ 3 generations, `C_max=3/8` attributed to Weinberg, the `≥3`-letter Pisot conjecture, Frobenius integrability — are recorded in BOOK_00 §00.9 (*Rejected forcing-links*). Each is refused on exactly the two admissible grounds (no second forced object exhibited / a named open gap), never by counting coincidences — the same blade, turned on the review.

### 05.8.S Deep-research is a candidate generator, not a source of truth

A growing share of D0's cross-domain links arrive from broad literature searches ("deep research"): an agent sweeps several fields and returns a list of theorems that *might* own a D0 link. This is valuable — but a deep-research report **systematically over-states status**. Its job is recall (find every candidate), not precision (certify each), so it labels coincidences "forcing" and analogies "bridge". Promoting those labels verbatim is an M1 violation at the *meta* level: it lets an external catalogue (the report) set D0's status field.

> **Verification rule (deep-research filter).** A deep-research result is a **candidate**, never a claim. Between the report and `theory_status_map.csv` there is a mandatory filter with two gates, and the status is set to the honest minimum that survives both:
> 1. **Mechanism gate.** Verify the *mechanism*, not the number. Two constructions yielding the same constant (e.g. `log φ` from a fusion rule and from a toral entropy) is a MECH-LIMIT until the *isomorphism* between them is exhibited — not a THE on the strength of the coincident value.
> 2. **Limits gate.** A match at one point is not an identity. If two quantities agree at a single parameter value but their limits diverge (e.g. `γ_Choptuik(D{=}4) ≈ 3/8` while `γ(D)` crosses and leaves `3/8`), the link is a **coincidence at that point**, recorded as HYP, never a derived bridge.

The honest landing is the *minimum* status that survives: a coincident number that lacks its mechanism is **MECH-LIMIT** (named gap = the missing map); a single-point match whose limits diverge is **HYP** (named gap = derive a common mechanism or accept the coincidence); an external uniqueness/classification theorem with a real D0-side anchor is a **BRIDGE-forcing** (cited owner, never core). Worked downgrades, kept as calibration: *Fibonacci `d_τ=φ` ⇒ `I_f=logφ`* — report FORCING, honest **MECH-LIMIT** while the categorical↔toral isomorphism is open, **closed at the entropy/finite-equivalence level** by the isomorphism (the fusion matrix `N_τ=[[0,1],[1,1]]` is the golden-mean SFT, `h_top = log` Perron `= logφ` on both the fusion and toral sides; full topological conjugacy stays the owned external residual, `D0-ADLER-WEISS-PARTITION-OWNER-001`) — the model case of a named gap closed by exhibiting the missing map, not by re-asserting the number; *`γ_Choptuik ↔ 3/8`* — report BRIDGE-owner, honest **HYP** (`D=4` coincidence, limits diverge); *Bekenstein `S≤A/4`* — report FORCING, honest **BRIDGE** (supports "capacity finite", does not force the D0 structure). Over-stating a deep-research status is the same sin as a fitted number written into a table — and is caught by the same firewall.

### 05.8.T An external contribution is a candidate for *closure*; reforge errors, do not discard

Where §05.8.S governs literature *searches*, this rule governs concrete *contributions* from other researchers (derivations, certs, proposed insertions). The constructive disposition is the dual of the deep-research filter: there the danger is over-statement to be downgraded; here the danger is **discarding a good idea because of a fixable error**. A contribution is a candidate for a *closure*, not a verdict to accept or reject wholesale.

> **Verification rule (external-contribution discipline).** Check the *mechanism* of each contribution and act by what the check reveals — never by the contributor's stated status:
> 1. **Where it closes** — develop it to its honest level (THE if the proof is complete, MECH-LIMIT if the mechanism is written but a map is open).
> 2. **Where it errs** — **reforge the error into a task**, do not discard the idea. An arithmetic or attribution error marks *where the work is not finished*, not where the idea is bad; the fix usually exposes a real clarification.
> 3. **Where it over-states** — bring it to the honest level **while keeping the valuable core**. Split the contribution into the part that holds (kept at its true status) and the part that does not (named gap / lower status). Never insert the over-statement as-is; never throw away the core with the overshoot.

Worked reforges, kept as calibration: *zone-matrix "char.poly ⇒ φ⁻¹"* (§02.18.4) — an **error** that, reforged, separated three conflated numbers (`spec(M)`, the `S_DE` window `eig(I−M)`, and the envelope tick `φ⁻¹`); *`α⁻¹` as a closed identity* (§02.13) — an **over-statement** whose core `ξ₅=φ⁻⁵` was kept THE while the α-line stayed CHK with `Δα` open; *`V_CKM U_PMNS^T=I`* (§06.30a) — an **error** reforged into the real quark–lepton complementarity `θ_C+θ_12≈45°` (HYP); *`I_f=Tr(\log T)/\mathrm{rank}`* (§09.03) — a wrong **formula** whose right number `log φ` was kept by fixing it to `h_{KS}=\log|\lambda_{\max}|`. In every case the contribution produced a closure or a clarification, never a discard — the constructive face of the same M1 discipline.

### 05.8.U Before "I derive X": grep the registry and the GOLDEN ledger first

The criterion of value is the **registry closure**, never the mere presence of something in a prior corpus. Two failure modes this rule blocks: (i) *re-deriving* what the registry already owns (a duplicate, or a weaker re-derivation that competes with the canonical owner); (ii) the false equation **"it is absent from the active corpus ⇒ it is valuable ⇒ add it"** — a result may be absent precisely *because the registry closes it better, holds it out of core, or marks it no-go*. So before opening any "I derive/return X": (1) grep `theory_status_map.csv` / `CLAIM_TO_LEAN_MAP.csv` for an existing owner — if found, *cross-reference* it, do not duplicate or compete; (2) grep `03_THEORY_MAP/GOLDEN_COVERAGE_LEDGER` — a GOLDEN result may already be present, or be a *definition without a derivation* (so "restoring" it would smuggle in an unproved step). Restore/return only what is **genuinely open AND not better-closed elsewhere AND not a no-go**; everything else is a cross-ref or is left alone. Worked calibration: `m_rest=m₀·W` is GOLDEN but its `W` are *undefined there* and the numeric mass spectrum is a deliberate no-go (`D0-GEN-MASS-001`), so `W` is kept a physical input, **not** core; the golden-split `W_int=W_ext²` is the already-CORE primitive `p+p²=1`, so it is cross-reffed, not re-derived.
