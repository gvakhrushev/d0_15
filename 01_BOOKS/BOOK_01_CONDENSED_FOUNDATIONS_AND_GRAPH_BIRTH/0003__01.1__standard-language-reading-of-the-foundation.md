### 01.1.3 The binary-fork proof method (the reusable forcing template)

Necessity of every Œ£ component is proved by one schema, the **binary fork** ‚Äî the standard three-branch M1 contradiction template that generates the individual results below [^b01-7]:

- **Branch A ‚Äî component absent:** show the set of distinguishable objects cannot be specified or fixed (realizability violated).
- **Branch B ‚Äî component present:** show it suffices to build the model.
- **Branch C ‚Äî component replaced by an alternative:** show the alternative demands an external catalog (‚ä•M1).

Branch A kills "do without it," branch C kills "swap it for something exogenous," branch B confirms sufficiency. The sections that follow are instances of this one template; this is the meta-method, kept here so the individual necessity proofs are not isolated facts.

### 01.1.4 Single code space `C*` is forced (LEM 1.1.5.2)

A realizable universe must have a single code ontology. ‚ä•-fork: suppose there is no single space ‚Äî two irreducible formats `C‚ÇÅ*`, `C‚ÇÇ*`. To compare `A ‚àà C‚ÇÅ*` with `B ‚àà C‚ÇÇ*`, `Œî(A,B)` is undefined (different types), so a translation mechanism `T : C‚ÇÅ* ‚Üî C‚ÇÇ*` is required.

- If `T` is an algorithm inside the theory, then `C_total = C‚ÇÅ ‚à™ C‚ÇÇ ‚à™ T` *is* the sought single space ‚Äî back to the thesis.
- If `T` is not an algorithm, it is an exogenous parameter set of correspondence rules ‚Äî a manual gluing of worlds ‚Äî ‚ä•M1.

Either way `C*` is single. Status: CORE-FORCING [^b01-8]. Existence of `Can` is not observer arbitrariness: in a finite alphabet under a fixed order ‚™Ø (length + lexicographic) the minimum of any non-empty equivalence class exists as a mathematical fact [^b01-9].

### 01.1.6 No hidden parameters (THE 1.1.11.B)

Either the parameter does not exist (canonized away by `D_min`) or it is part of the code (written into History `R`). There are no invisible parameters steering the world past History.
