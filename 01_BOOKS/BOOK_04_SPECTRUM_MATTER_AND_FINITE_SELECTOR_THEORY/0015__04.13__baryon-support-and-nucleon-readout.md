**Honest scope (this is a HYP row, not a forced selector).** "Unique zero-residual over `Fin 613`" defeats the *±1-window* critic (a critic cannot move the value by `±1` while keeping the score), but it does **not** defeat the *score-construction* critic: the score functional's reference data were themselves read off the proton benchmark, so the zero sits at `306` partly by construction. Per the corpus's own rule (BOOK_05 §05.8a, "a number that *fits* is not a number that is *forced*"), this row is **HYP**, the same honest status the charged-lepton decimals carry in §04.8 — not a forced CORE selector. A genuine forcing would need a score whose reference data are fixed *independently* of the benchmark, plus a reachable FAIL control against score reverse-construction.

Earlier drafts used nearest-neighbour windows such as `{305,306,307}` and `{18,19,20}` as a quick anti-retuning check. Those windows are no longer the active theory. The active owners are:

```text

```

The electroweak radial depth is selected over `Fin 71`, the proton destructive readout over `Fin 613`, and the beta weak-unlock depth over `Fin 39`.  This closes only the **±1-window** freedom: a critic cannot move the selected value by leaving the local window while keeping the same score.  It does **not** close the **score-construction** freedom — choosing a different score functional is exactly the remaining handle, and the present scores' reference data were read off the benchmarks — so these full-support selectors remain **HYP**, not forced. Promotion to a forced selector requires a score whose reference data are fixed independently of the benchmark plus a reachable FAIL control against reverse-construction.
